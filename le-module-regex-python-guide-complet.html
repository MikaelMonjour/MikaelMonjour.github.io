Le module regex en Python : le guide complet pour aller au-delà de re

Published: mer. 05 mars 2026
By Mikael Monjour

In tech.

tags: python regex data tech

Vous nettoyez des données dans un CRM. Des milliers de numéros de téléphone, d'adresses e-mail et de noms de sociétés — chacun saisi à la main par un commercial pressé. +33 6 12 34 56 78, 06-12-34-56-78, 0612345678 : trois écritures pour le même numéro. Votre mission : tout normaliser en un seul format.

Avec le module standard re de Python, vous y arrivez — tant que les cas restent simples. Mais dès que les patterns deviennent complexes (du texte approximatif, des jeux de caractères Unicode, des groupes récursifs), re atteint ses limites.

C'est là qu'entre en jeu le module regex — une alternative rétrocompatible avec re, mais avec des fonctionnalités que re n'a tout simplement pas.

Ce guide couvre tout ce que le module regex apporte de plus que re, avec des explications simples et des exemples concrets.

Avant de commencer : les bases

Quelques termes utilisés dans cet article :

  • Regex (expression régulière) : un motif de texte qui décrit un ensemble de chaînes de caractères. Par exemple, \d{3} correspond à n'importe quel nombre de 3 chiffres (123, 456, 789...).
  • Pattern : le motif qu'on cherche dans le texte. C'est la regex elle-même.
  • Match : quand le texte correspond au pattern. "Le texte matche le pattern" = "le texte correspond au motif".
  • Flag : une option qui modifie le comportement de la regex. Par exemple, le flag IGNORECASE rend la recherche insensible aux majuscules/minuscules.
  • Groupe de capture : une partie du pattern entourée de parenthèses (). Permet d'extraire une sous-partie du texte trouvé.
  • GIL (Global Interpreter Lock) : un mécanisme interne de Python qui empêche deux threads d'exécuter du code Python en même temps. Le module regex peut le libérer pendant la recherche, ce qui accélère le traitement en parallèle.

Installation

pip install regex

Ensuite, dans votre code :

import regex

# Il remplace re — même syntaxe, mêmes méthodes
result = regex.search(r'\d+', 'Il y a 42 pommes')
print(result.group())  # 42

Tout ce qui fonctionne avec re fonctionne avec regex. Vous pouvez remplacer import re par import regex sans rien casser.

Version 0 vs Version 1 : deux comportements

Le module regex propose deux modes de fonctionnement :

  • Version 0 (V0) : compatible avec le module re standard. C'est le mode par défaut. Si vous avez du code existant qui utilise re, il fonctionnera à l'identique en V0.
  • Version 1 (V1) : active les fonctionnalités avancées (ensembles imbriqués, meilleure gestion des correspondances de largeur zéro, flags scopés). C'est le mode recommandé pour les nouveaux projets.

Pour activer la V1, deux options :

# Via un flag dans le code
regex.search(pattern, text, flags=regex.V1)

# Ou directement dans le pattern
regex.search(r'(?V1)mon_pattern', text)

La principale différence entre les deux versions concerne les correspondances de largeur zéro — c'est-à-dire les cas où le pattern correspond à "rien" (un espace vide entre deux caractères). En V0, ces cas sont parfois mal gérés. En V1, le comportement est corrigé :

# V0 : comportement de re (ancien)
regex.sub('(?V0).*', 'x', 'test')
# Résultat : 'x'

# V1 : comportement corrigé
regex.sub('(?V1).*', 'x', 'test')
# Résultat : 'xx'

Les correspondances insensibles à la casse en Unicode

En français et dans beaucoup de langues, la gestion des majuscules/minuscules est plus complexe qu'en anglais. Par exemple, en allemand, le caractère ß (eszett) équivaut à SS en majuscules.

Le module regex gère ces cas grâce au full case-folding (conversion complète des majuscules/minuscules) :

# En V1, le full case-folding est activé par défaut
regex.match(r"(?iV1)strasse", "straße").span()
# Résultat : (0, 6) — ça matche !

regex.match(r"(?iV1)straße", "STRASSE").span()
# Résultat : (0, 7) — dans l'autre sens aussi

Le flag FULLCASE (ou F) contrôle ce comportement. En V1, il est activé par défaut. En V0, il est désactivé pour rester compatible avec re.

Ensembles imbriqués et opérations sur les ensembles

C'est l'une des fonctionnalités les plus puissantes du module regex. En V1, vous pouvez combiner des ensembles de caractères avec des opérations logiques :

  • || — union : "x ou y"
  • && — intersection : "x et y"
  • -- — différence : "x mais pas y"
  • ~~ — différence symétrique : "x ou y, mais pas les deux"

Exemple concret : vous voulez extraire toutes les lettres minuscules sauf les voyelles :

# Toutes les lettres a-z SAUF les voyelles
regex.findall(r'(?V1)[[a-z]--[aeiou]]', 'bonjour le monde')
# Résultat : ['b', 'n', 'j', 'r', 'l', 'm', 'n', 'd']

Autre cas utile : extraire tous les caractères qui sont à la fois ASCII et des lettres :

regex.findall(r'(?V1)[\p{ASCII}&&\p{Letter}]', 'café 123 naïf')
# Les lettres ASCII uniquement (pas les é, ï)

Les flags : scopés vs globaux

Un flag modifie le comportement d'une regex. Le module regex distingue deux types :

  • Flags scopés (portée limitée) : ils s'appliquent à une partie du pattern et peuvent être activés ou désactivés. Exemples : IGNORECASE, DOTALL, VERBOSE, FULLCASE, MULTILINE, WORD.
  • Flags globaux (tout le pattern) : ils s'appliquent à l'ensemble du pattern et peuvent seulement être activés. Exemples : ASCII, UNICODE, REVERSE, BESTMATCH, ENHANCEMATCH, POSIX, VERSION0, VERSION1.

L'intérêt des flags scopés, c'est de pouvoir appliquer un comportement à une sous-partie du pattern seulement :

# IGNORECASE seulement sur la première partie
regex.search(r'(?i:hello) world', 'HELLO world')
# Matche — "HELLO" est insensible à la casse, mais "world" doit être en minuscules

Le fuzzy matching : la correspondance approximative

C'est probablement la fonctionnalité la plus utile pour le nettoyage de données. Le fuzzy matching permet de trouver des correspondances même quand le texte contient des fautes de frappe.

Trois types d'erreurs sont gérés :

  • i — insertion : un caractère en trop
  • d — suppression : un caractère manquant
  • s — substitution : un caractère remplacé par un autre
  • e — n'importe quel type d'erreur

La syntaxe : on ajoute {erreurs} après l'élément à rendre flou :

# Chercher "bonjour" avec au maximum 1 erreur
regex.search(r'(bonjour){e<=1}', 'bonjoure le monde')
# Matche "bonjoure" (1 insertion)

# Chercher "bonjour" avec au maximum 2 erreurs de n'importe quel type
regex.search(r'(bonjour){e<=2}', 'bonjuor le monde')
# Matche "bonjuor" (1 substitution + potentiellement 1 autre erreur)

Cas concret en growth : vous avez une liste de noms d'entreprises dans votre CRM et vous voulez dédupliquer. "Microsft", "Microsoftt", "Microsoft" — c'est la même boîte :

import regex

companies = ['Microsoft', 'Microsft', 'Microsoftt', 'Google', 'Gogle', 'Googl']

for name in companies:
    m = regex.fullmatch(r'(?:Microsoft){e<=1}', name)
    if m:
        print(f'{name} → Microsoft (erreurs: {m.fuzzy_counts})')

# Microsft → Microsoft (erreurs: (0, 0, 1))   # 1 suppression
# Microsoftt → Microsoft (erreurs: (0, 1, 0)) # 1 insertion
# Microsoft → Microsoft (erreurs: (0, 0, 0))  # correspondance exacte

L'objet match contient deux attributs utiles :

  • fuzzy_counts — un tuple (substitutions, insertions, suppressions) qui donne le nombre d'erreurs.
  • fuzzy_changes — un tuple qui donne les positions exactes des erreurs dans le texte.

Deux flags améliorent le fuzzy matching :

  • ENHANCEMATCH — essaie d'améliorer la qualité du match trouvé (réduire le nombre d'erreurs).
  • BESTMATCH — cherche le meilleur match possible au lieu du premier trouvé.
# Sans ENHANCEMATCH : le premier match trouvé
regex.search("(dog){e}", "cat and dog")[1]
# Résultat : 'cat' — car "cat" matche "dog" avec 3 substitutions (illimité)

# Avec une limite d'erreurs
regex.search("(dog){e<=1}", "cat and dog")[1]
# Résultat : ' dog' (avec un espace devant) — 1 suppression

# Avec ENHANCEMATCH : on affine le résultat
regex.search("(?e)(dog){e<=1}", "cat and dog")[1]
# Résultat : 'dog' (sans espace) — correspondance améliorée

On peut aussi assigner des coûts à chaque type d'erreur :

# Chaque insertion coûte 2, chaque suppression coûte 2,
# chaque substitution coûte 1, coût total max de 4
regex.search(r'(bonjour){2i+2d+1s<=4}', texte)

Les patterns récursifs

Un pattern récursif est un pattern qui peut se référencer lui-même. C'est utile pour analyser des structures imbriquées comme du JSON, du HTML ou des expressions mathématiques avec des parenthèses.

# (?R) essaie de matcher l'ensemble du pattern récursivement
regex.match(r"(Tarzan|Jane) loves (?1)", "Tarzan loves Jane").groups()
# Résultat : ('Tarzan',)
# (?1) réutilise le sous-pattern du groupe 1 : (Tarzan|Jane)

# Détecter des palindromes
m = regex.search(r"(\w)(?:(?R)|(\w?))\1", "kayak")
m.group(0)
# Résultat : 'kayak'

En clair : (?1), (?2), etc. réutilisent le sous-pattern d'un groupe de capture. (?R) ou (?0) réutilisent le pattern entier. (?&nom) réutilisent un groupe nommé.

DEFINE : déclarer des sous-patterns réutilisables

Le bloc (?(DEFINE)...) permet de déclarer des sous-patterns sans qu'ils ne correspondent à rien par eux-mêmes. Ils sont ensuite appelés ailleurs dans le pattern :

# Déclarer un pattern "prix" réutilisable
pattern = regex.compile(r'''(?x)
    (?(DEFINE)
        (?P<price>\$\d+\.\d{2}\b)
    )
    (?&price)[-+](?&price)=(?&price)
''')

pattern.search('total: $1.99+$2.99=$4.98')
# Matche "$1.99+$2.99=$4.98"

L'avantage : au lieu de répéter \$\d+\.\d{2}\b trois fois dans le pattern (ce qui le rend illisible), on le déclare une fois sous le nom "price" et on le réutilise avec (?&price).

Les captures multiples

Avec le module standard re, quand un groupe de capture est répété (par exemple (\w+)+), seule la dernière capture est conservée. Le module regex conserve toutes les captures :

m = regex.search(r"(\w{3})+", "123456789")

# re ne garderait que la dernière capture :
m.group(1)
# Résultat : '789'

# regex conserve TOUTES les captures :
m.captures(1)
# Résultat : ['123', '456', '789']

# Positions de début et de fin de chaque capture :
m.starts(1)   # [0, 3, 6]
m.ends(1)     # [3, 6, 9]
m.spans(1)    # [(0, 3), (3, 6), (6, 9)]

Avec capturesdict, on récupère un dictionnaire complet des captures par nom de groupe :

m = regex.match(
    r"(?:(?P<word>\w+) (?P<digits>\d+)\n)+",
    "one 1\ntwo 2\nthree 3\n"
)

m.groupdict()
# {'word': 'three', 'digits': '3'} — seulement la dernière capture

m.capturesdict()
# {'word': ['one', 'two', 'three'], 'digits': ['1', '2', '3']} — tout !

Les correspondances partielles

Les correspondances partielles (partial matches) sont utiles quand vous validez une saisie en temps réel — par exemple, un formulaire où l'utilisateur tape un numéro de carte bancaire chiffre par chiffre :

pattern = regex.compile(r'\d{4}')

# L'utilisateur n'a encore rien tapé :
pattern.fullmatch('', partial=True)
# Match partiel — c'est encore possible

# L'utilisateur tape une lettre :
pattern.fullmatch('a', partial=True)
# None — ça ne correspondra jamais

# L'utilisateur tape un chiffre :
pattern.fullmatch('1', partial=True)
# Match partiel — en bonne voie

# Trois chiffres :
pattern.fullmatch('123', partial=True)
# Match partiel — presque

# Quatre chiffres :
pattern.fullmatch('1234', partial=True)
# Match complet !

# Cinq chiffres :
pattern.fullmatch('12345', partial=True)
# None — trop long

L'objet match a un attribut partial qui vaut True si c'est un match partiel et False si c'est un match complet.

Le matching POSIX : le plus long match d'abord

Par défaut, les regex retournent le premier match trouvé. Avec le flag POSIX ((?p)), le module cherche le match le plus long possible à la position la plus à gauche — c'est la norme POSIX :

regex.search(r'(?p)one(self)?(selfsufficient)?', 'oneselfsufficient')
# Résultat : 'oneselfsufficient' — le match le plus long

Attention : le matching POSIX est plus lent car il ne s'arrête pas au premier résultat. Il continue à chercher pour s'assurer qu'il n'y a pas de match plus long.

Les groupes atomiques et quantificateurs possessifs

Un groupe atomique (?>...) empêche le moteur de regex de revenir en arrière (backtracking) dans le sous-pattern. Si le reste du pattern échoue, le groupe atomique échoue en bloc au lieu de tester d'autres possibilités.

Les quantificateurs possessifs (?+, *+, ++) font la même chose pour les répétitions : ils ne rendent jamais ce qu'ils ont consommé.

En pratique, ces deux fonctionnalités permettent d'éviter des regex qui prennent un temps exponentiel sur certains textes (le fameux "catastrophic backtracking") :

# Sans groupe atomique : peut être très lent sur de longues chaînes
regex.search(r'(a+)+b', 'aaaaaaaaaaac')

# Avec groupe atomique : échoue immédiatement
regex.search(r'(?>a+)+b', 'aaaaaaaaaaac')

Autres fonctionnalités utiles

\K — garder seulement une partie du match

\K réinitialise le début du match. Tout ce qui est avant \K dans le pattern doit correspondre, mais n'est pas inclus dans le résultat :

m = regex.search(r'(\w\w\K\w\w\w)', 'abcdef')
m[0]   # 'cde' — le match commence après \K
m[1]   # 'abcde' — mais le groupe capture tout

Les listes nommées : \L<nom>

Au lieu de construire un pattern avec des alternatives séparées par |, vous pouvez passer une liste directement :

options = ["premier", "deuxième", "troisième", "quatrième"]
p = regex.compile(r"\L", options=options)

p.search("le deuxième choix")
# Matche "deuxième"

L'avantage : le module gère automatiquement l'ordre et l'échappement des caractères spéciaux.

Lookbehind de longueur variable

Le module re exige que les lookbehinds (assertions arrière) aient une longueur fixe. Le module regex n'a pas cette limitation :

# Impossible avec re, possible avec regex :
regex.search(r'(?<=\d+\.)\d+', '3.14')
# Matche '14' — lookbehind de longueur variable

Multithreading

Le module regex libère le GIL (verrou global de Python) pendant les recherches sur des chaînes immuables. Cela permet à d'autres threads de s'exécuter en parallèle, ce qui peut accélérer le traitement de gros volumes de texte :

# Forcer la libération du GIL
regex.search(pattern, text, concurrent=True)

Ancre de recherche \G

\G correspond à la position où la recherche précédente s'est terminée. Utile pour des correspondances contiguës :

regex.findall(r"\w{2}", "abcd ef")
# Résultat : ['ab', 'cd', 'ef']

regex.findall(r"\G\w{2}", "abcd ef")
# Résultat : ['ab', 'cd'] — s'arrête à l'espace car \G exige la contiguïté

Propriétés Unicode

Le module supporte les propriétés Unicode pour des recherches très précises :

# Toutes les lettres du script latin
regex.findall(r'\p{Script=Latin}+', 'café こんにちは hello')
# Résultat : ['café', 'hello']

# Tous les chiffres, toutes écritures confondues
regex.findall(r'\p{Number}+', 'prix: 42€ ou ٤٢ dinars')

# Classes POSIX
regex.findall(r'[[:alpha:]]+', 'hello 123 world')

Récapitulatif : quand utiliser regex au lieu de re

Voici les situations où le module regex justifie le remplacement de re :

Recherches simples (email, téléphone, URL)OuiOui
Fuzzy matching (fautes de frappe)NonOui
Patterns récursifs (JSON, HTML imbriqué)NonOui
Opérations sur les ensembles de caractèresNonOui
Captures multiples d'un groupe répétéNonOui

Checklist : migrer de re à regex

  1. Installer le module : pip install regex
  2. Remplacer l'import : import regex au lieu de import re. Tout votre code existant continue de fonctionner.
  3. Tester en V0 d'abord : vérifiez que rien ne casse avec le comportement par défaut (compatible re).
  4. Passer en V1 progressivement : activez (?V1) dans les patterns qui ont besoin des fonctionnalités avancées.
  5. Utiliser le fuzzy matching pour le nettoyage de données (CRM, déduplication, normalisation de texte).
  6. Utiliser les captures multiples quand vous parsez des structures répétées (logs, CSV, données semi-structurées).
  7. Utiliser les groupes atomiques si vous avez des regex lentes sur certaines entrées (catastrophic backtracking).
  8. Documenter les changements : notez dans votre code pourquoi vous utilisez regex au lieu de re, pour que vos collègues comprennent la dépendance.

Le module regex est un remplacement transparent de re avec des fonctionnalités que le module standard n'a tout simplement pas. Si vous travaillez avec de la donnée texte — nettoyage de CRM, parsing de logs, extraction d'informations, validation de saisie — il devrait faire partie de votre boîte à outils.
